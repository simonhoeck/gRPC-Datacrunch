Die Bond Analyzer – Setup

Schritt-für-Schritt Anleitung für den Zielrechner
1
Python installieren
Python 3.10 oder neuer wird benötigt. Im Browser öffnen: 
https://www.python.org/downloads/
ℹ Bei der Installation "Add Python to PATH" aktivieren.
Version prüfen:
python --version
2
Projektdateien kopieren
Das Repository von GitHub klonen oder den grpc/ Ordner direkt auf den Rechner kopieren. 
# Option A – Git (falls installiert)
git clone https://github.com/simonhoeck/gRPC-Datacrunch.git
cd gRPC-Datacrunch\grpc
# Option B – nur in den grpc-Ordner wechseln (falls manuell kopiert)
cd C:\Pfad\zum\grpc-Ordner
3
Python-Pakete installieren
Einmalig ausführen (benötigt Internetzugang):
pip install flask pymongo python-dotenv
✓ Falls pip nicht gefunden: python -m pip install flask pymongo python-dotenv
4
.env Datei anlegen
Im grpc/ Ordner eine Datei namens .env erstellen (keine Dateiendung!) mit folgendem Inhalt:
grpc\.env
MONGO_HOST=localhost
MONGO_PORT=27017
MONGO_USER=admin
MONGO_PASSWORD=SecurePasswort123
MONGO_DB=grpcdata
PORT=5001
⚠ Die Datei heißt .env – kein Dateiname davor, nur die Endung. Im Windows-Explorer ggf. "Dateinamen-Erweiterungen anzeigen" aktivieren.
5
Server starten
Eine Eingabeaufforderung (cmd) im grpc/ Ordner öffnen und: 
python server.py
Erwartete Ausgabe:
  Die Bond Analyzer
  --------------------------------------
  URL      : http://localhost:5001
  MongoDB  : ATRAPC0114:27017  /  DB: grpcdata
  Mode     : JSON files  +  live MongoDB API
  --------------------------------------
 * Running on http://0.0.0.0:5001
ℹ Das Fenster muss offen bleiben solange die Anwendung läuft. Mit Strg+C beenden.
6
Browser öffnen
Im Browser (Chrome, Edge, Firefox) aufrufen:
http://localhost:5001
✓ Die Seite lädt automatisch die JSON-Dateien aus dem grpc/ Ordner. Für Live-Daten aus MongoDB oben rechts auf MongoDB klicken.
Schnell-Checkliste
  • Python 3.10+ installiert, im PATH
  • pip install flask pymongo python-dotenv ausgeführt
  • .env Datei im grpc/ Ordner vorhanden
  • server.py läuft (cmd-Fenster offen)
  • MongoDB Docker-Container läuft (docker compose up -d im gRPC-Ordner)
  • Browser: http://localhost:5001
Häufige Probleme
Port bereits belegt
In .env die Zeile PORT=5001 auf einen anderen Wert ändern (z.B. 5002).
MongoDB nicht verbunden (roter Punkt)
Netzwerkverbindung zu ATRAPC0114 prüfen. JSON-Modus funktioniert auch ohne MongoDB.
ModuleNotFoundError
pip-Befehl aus Schritt 3 noch einmal ausführen.
.env wird nicht gefunden
cmd muss im selben Ordner wie server.py geöffnet sein.
